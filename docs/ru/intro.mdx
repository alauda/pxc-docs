# Введение

**Служба базы данных Alauda для MySQL-PXC** — это нативное решение для Kubernetes, предназначенное для упрощения развертывания, управления и масштабирования кластеров MySQL, построенных на Percona XtraDB Cluster (PXC). Оператор использует возможности оркестрации Kubernetes для автоматизации критически важных задач управления базами данных, включаяProvisioning кластера, резервное копирование, переключение при сбое и масштабирование.

**Percona XtraDB Cluster (PXC)** — это решение MySQL с открытым исходным кодом, уровня предприятий, предназначенное для высокой доступности и согласованности данных. Оно использует синхронную репликацию, чтобы гарантировать, что данные являются согласованными на всех узлах кластера. PXC обеспечивает устойчивость к сбоям, автоматическое переключение при сбое и масштабируемость, что делает его идеальным для работы высокодоступных баз данных MySQL в критически важных средах.

Это создает основу для оператора Percona для MySQL, позволяя упростить развертывание и управление Percona XtraDB Cluster в средах Kubernetes.

---

## Ключевые особенности

#### 1. **Автоматизированное развертывание и масштабирование**

- Упрощает создание кластеров MySQL с минимальной конфигурацией.
- Динамически масштабирует экземпляры в зависимости от требований к рабочей нагрузке, оптимизируя использование ресурсов.

#### 2. **Высокая доступность**

- Обеспечивает нулевое время простоя с автоматическими механизмами переключения при сбое.
- Использует синхронную репликацию для поддержания согласованности данных между узлами.

#### 3. **Самоисцеление**

- Автоматически обнаруживает и восстанавливается от сбоя узлов для поддержания здорового состояния кластера.
- Обеспечивает непрерывность работы с минимальным вмешательством человека.

#### 4. **Резервное копирование и восстановление**

- Обеспечивает согласованные, автоматизированные резервные копирования в облачное хранилище или на локальные диски.
- Предоставляет возможность быстрого восстановления, гарантируя безопасность данных и непрерывность бизнеса.

#### 5. **Улучшенная безопасность**

- Поддерживает шифрование данных в состоянии покоя и во время передачи.
- Интегрируется с контролем доступа на основе ролей (RBAC) Kubernetes для безопасного выполнения операций с базами данных.

#### 6. **Упрощение операций**

- Обеспечивает бесшовную интеграцию с инструментами, нативными для Kubernetes, такими как [`kubectl`](https://kubernetes.io/docs/reference/kubectl/).
- Оптимизирует мониторинг, управление и устранение неполадок баз данных.

#### 7. **Гибкость для облачно-ориентированных архитектур**

- Оптимизировано для публичных, частных и гибридных облачных развертываний.
- Позволяет объединённое управление базами данных в различных средах.
