# Остановка и запуск экземпляра

Функция однокнопочной остановки и запуска отвечает повседневным потребностям в разработке и обслуживании, позволяя вам быстро приостановить или возобновить экземпляры промежуточного программного обеспечения. В сценариях, таких как запланированное время простоя, высокая нагрузка на узел и окружения для разработки и тестирования, вам может потребоваться временно освободить ресурсы CPU и памяти без удаления экземпляров. Функция однокнопочной остановки и запуска помогает сэкономить ресурсы и гибко управлять экземплярами, тем самым снижая затраты и повышая эффективность.

## Предварительные требования

Для остановки экземпляра требуется, чтобы экземпляр находился в состоянии **Обработка**, **Запуск**, **Аномальный** или **Неизвестный**.

Для запуска экземпляра требуется, чтобы экземпляр находился в состоянии **Остановка** или **Остановленный**.

## Процедура

<Tabs>
  <Tab label="CLI">
    #### Остановка экземпляра

    ```bash
    kubectl patch mysql -n <namespace> <name> --type='merge' --patch '
    spec:
        pause: true
    '
    ```

    #### Запуск экземпляра

    ```bash
    kubectl patch mysql -n <namespace> <name> --type='merge' --patch '
    spec:
        pause: false
    '
    ```
  </Tab>

  <Tab label="Веб-консоль">
    #### Остановка экземпляра

    1. Нажмите на **MySQL-PXC** в левой панели навигации.

    2. Нажмите на ***Имя пространства имен***.

    3. Нажмите на ***Имя экземпляра***.

    4. Нажмите на **Операции** > **Остановить** и подтвердите.

    ### Доступные функции после остановки

    | Функция                    | Описание                                                |
    | -------------------------- | ------------------------------------------------------- |
    | **Консоль терминала**      | Консоль терминала недоступна.                          |
    | **Операции**               | Невозможно перезапустить, обновить или изменить конфигурацию расписания. |
    | **Топология**              | Страница топологии недоступна для просмотра.           |
    | **Конфигурация параметров** | Невозможно обновить параметры или экспортировать параметры. |
    | **Управление пользователями** | Управление пользователями невозможно.                 |
    | **Резервное копирование и восстановление** | Резервное копирование не может быть выполнено.   |
    | **Инспекция**              | Результаты инспекции для текущего экземпляра недействительны. |

    #### Запуск экземпляра

    1. Нажмите на **MySQL-PXC** в левой панели навигации.

    2. Нажмите на ***Имя пространства имен***.

    3. Нажмите на ***Имя экземпляра***.

    4. Нажмите на **Операции** > **Запустить** и подтвердите.
  </Tab>
</Tabs>
