# Перезапуск экземпляра

Если экземпляр сталкивается с эксплуатационными неисправностями, избыточным количеством подключений или другими неизвестными проблемами, которые невозможно идентифицировать в процессе его работы, проблемы можно решить, перезапустив экземпляр.

## Предварительные условия

Экземпляр должен находиться в состоянии "Запущен".

**Примечание:**

В процессе перезапуска Pods будут перезапускаться по одному, что может вызвать временные перебои в обслуживании.

Если экземпляр находится в состоянии "Обработка", это означает, что экземпляр может нормально запускаться; пожалуйста, проявите терпение.

Если экземпляр не удается запустить или он ненормальный, то есть экземпляр находится в состоянии "Обработка" или "Ошибка", он автоматически перезапустится после того, как истечет время проверки работоспособности; пожалуйста, проявите терпение.

## Процедура

<Tabs>
  <Tab label="CLI">
    1. Перезапустите экземпляр

    ```bash
    kubectl patch mysql -n <namespace> <name> --type='merge' --patch '
    spec:
        pxc:
          pxc:
              annotations:
                  percona.com/Restart-time: '$(date -u +'%Y-%m-%dT%H:%M:%SZ')'
          proxysql:
              annotations:
                  percona.com/Restart-time: '$(date -u +'%Y-%m-%dT%H:%M:%SZ')'    
    '
    ```
  </Tab>

  <Tab label="Web Console">
    1. Нажмите на **MySQL-PXC** в левой панели навигации.

    2. Нажмите на ***Имя пространства имен***.

    3. Нажмите на ***Имя экземпляра***.

    4. Нажмите **Действия** > **Перезапуск**, и подтвердите.
  </Tab>
</Tabs>
