# Мониторинг

Встроенные в панель данные мониторинга могут использоваться для мониторинга MySQL и уведомлений по ресурсам, производительности и пропускной способности, а также поддерживают настройку уведомлений.

Интуитивно представленные данные мониторинга могут быть использованы для поддержки принятия решений по оперативной проверке или тонкой настройке производительности.

### Панель мониторинга производительности

Платформа по умолчанию собирает общепринятые метрики мониторинга, связанные с ресурсами, производительностью и пропускной способностью MySQL. На вкладке **Мониторинг** экземпляра можно просмотреть данные мониторинга в реальном времени для этих метрик.

| Метрика                     | Метрика мониторинга по умолчанию                                                                                                                                                                                                                      |
| --------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Использование CPU Pod       | Использование CPU пода. Под с использованием CPU выше 100% будет проигнорирован, потому что в нормальных условиях использование CPU не превышает 100%. Если использование превышает 100%, проверьте, установлены ли лимиты ресурсов для всех контейнеров пода. |
| Использование памяти Pod     | Использование памяти пода. Под с использованием памяти выше 100% будет проигнорирован, потому что в нормальных условиях использование памяти не превышает 100%. Если использование превышает 100%, проверьте, установлены ли лимиты ресурсов для всех контейнеров пода. |
| Полученные данные сети      | Среднее количество байтов в секунду, полученное из сети за последние 5 минут                                                                                                                                                                          |
| Переданные данные сети      | Среднее количество байтов в секунду, отправленных в сеть за последние 5 минут                                                                                                                                                                          |
| Использование PVC           | Использование хранилища PVC.                                                                                                                                                                                                                           |
| Соединения MySQL           | Max Connections - максимальное разрешенное количество одновременных клиентских соединений. По умолчанию это 151. Увеличение этого значения увеличивает количество дескрипторов файлов, необходимых mysqld.                                             |
| Активность потоков клиента MySQL | Threads Connected - количество открытых соединений, в то время как Threads Running - количество потоков, которые не спят.                                                                                                                                     |
| Блокировки таблиц MySQL     | MySQL использует несколько различных блокировок по разным причинам. На этом графике мы можем увидеть, сколько блокировок на уровне таблицы запросил MySQL у механизма хранения.                                                                            |
| Сортировки MySQL           | Этот график также показывает, когда сортировки должны были просканировать всю таблицу или заданный диапазон таблицы, чтобы вернуть результаты, которые не могли быть отсортированы через индекс.                                                          |
| Медленные запросы MySQL    | Медленные запросы определяются как запросы, которые медленнее, чем установленное значение long\_query\_time.                                                                                                                                              |
| Запросы к буферному пулу InnoDB | Запросы к буферному пулу InnoDB относятся к общему количеству операций чтения и записи, сделанных к буферному пулу InnoDB.                                                                                                                                 |
| TPS                          | Transactions Per Second (TPS) - это ключевая метрика производительности, которая измеряет количество транзакций, которые MySQL обрабатывает в секунду.                                                                                                    |
| Команды                     | Команды относятся к количеству SQL операторов, выполненных MySQL, включая запросы, вставки, обновления, удаления и другие операции с базой данных.                                                                                                       |
