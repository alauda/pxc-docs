# Журнал

Журналы предоставляют ценную информацию. Имеет смысл проверять журналы Pods базы данных и Pod оператора. Следующие флаги полезны для проверки журналов с помощью команды kubectl logs.

## Процедура

<Tabs>
  <Tab label="CLI">
    1. Проверьте все связанные Pods экземпляра, включая pxc pod и proxysql pod.

    ```bash
    kubectl get pod -n <namespace> -l "middleware.instance/name=<name>"
    ```

    2. Просмотрите журналы запросов для конкретного pod

    ```bash
    kubectl logs -n <namespace> <pod>
    ```
  </Tab>

  <Tab label="Web Console">
    ### Операции в интерфейсе

    1. В левом навигационном меню нажмите **MySQL-PXC**.

    2. Нажмите на ***Имя пространства имен***.

    3. Нажмите на ***Имя экземпляра*** .

    4. На вкладке **Realtime Log** установите критерии поиска.

       - Чтобы найти журналы конкретных Pods, которые являются репликами MySQL: В разделе **Pods** выберите pod и нажмите **Поиск**.

    5. Подождите немного и просмотрите результаты.
  </Tab>
</Tabs>
