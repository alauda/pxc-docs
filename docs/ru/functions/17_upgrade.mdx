# Стратегия обновления патч-версии

С выходом исправлений уязвимостей или новых функций сервис базы данных Alauda для MySQL-PXC будет выполнять обновления патч-версий. Во время этих обновлений патч-версий группы контейнеров будут перезапускаться по очереди. Рекомендуется выбирать подходящее время для обновления. Когда появится обновляемая патч-версия, рекомендуется обновлять как можно скорее, чтобы применить последние исправления к компонентах.

## Процедура

<Tabs>
  <Tab label="веб Консоль">
    1. Перейдите на страницу деталей экземпляра.

    2. Нажмите **Стратегия обновления патч-версии**.

    3. Выберите **Ручной** или **Автоматический** по мере необходимости.

    4. Нажмите **Обновить**, чтобы завершить настройку.

    ## Обновление патч-версии

    ### Предварительные условия

    Когда доступна обновляемая патч-версия, значок обновления будет отображаться в списке экземпляров, и появится сообщение об обновлении на странице деталей экземпляра.

    ### Процедура

    1. Перейдите на страницу деталей экземпляра.

    2. Нажмите **Обновить сейчас** рядом с сообщением об обновлении.

    3. Прочитайте сообщение об обновлении, чтобы понять изменения версий компонентов и плагинов.

    4. Нажмите **Обновить сейчас**.

    5. Дождитесь изменения статуса экземпляра на работающий. Обновление прошло успешно.

    ## Пакетное обновление патч-версии

    ### Предварительные условия

    У пользователя должны быть права администратора платформы.

    ### Процедура

    1. В левой навигационной панели нажмите **Операторы**.

    2. Нажмите **оператор**, чтобы войти на страницу деталей оператора.

    3. Выберите экземпляры для обновления и нажмите **Пакетное обновление**.

    4. Прочитайте сообщение об обновлении, чтобы понять изменения версий компонентов и плагинов.

    5. Нажмите **Обновить сейчас**.

    6. Дождитесь изменения статусов экземпляров на работающие. Обновление прошло успешно.
  </Tab>
</Tabs>
