# 部署

## 安装 Alauda 应用服务基础插件

::: tip 可选部署
该插件是应用服务图形界面的基本组成部分；如果不需要 Web 控制台，可以省略此插件。
:::

### 前提条件

1. 下载与您的平台匹配的 Alauda 应用服务基础安装包。
2. 使用平台的应用列表功能在 ***global cluster*** 中列出 Alauda 应用服务基础。

### 执行步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 在左侧导航栏中，转到 **应用商店管理** > **集群插件**。
    2. 选择 **global** 集群。
    3. 单击 **Alauda 应用服务基础** 插件旁边的操作按钮 > **部署**。
  </Tab>
</Tabs>

## 安装 Alauda 应用服务 RDS 框架插件

::: tip 可选部署
该插件是应用服务图形界面的基本组成部分；如果不需要图形服务，可以省略此插件。
:::

### 前提条件

1. 下载与您的平台匹配的 Alauda 应用服务 RDS 框架安装包。
2. 使用平台的应用列表功能在任何希望使用应用服务图形功能的集群中列出 Alauda 应用服务 RDS 框架。

### 执行步骤

<Tabs>
  <Tab label="Web 控制台">
    1. 登录平台，转到 **平台管理** 页面。
    2. 在左侧导航栏中，选择 **应用商店管理** -> **操作员** 进入 **操作员中心** 页面。
    3. 找到 **Alauda 应用服务 RDS 框架**，单击 **部署**，进入部署页面。

    配置参数：

    | **参数**              | **推荐配置**                                                                                                                          |
    | :--------------------- | :------------------------------------------------------------------------------------------------------------------------------------ |
    | **升级策略**          | **手动**：当操作员中心有新版本时，需要手动确认以将操作员升级到最新版本。                                                             |
    | **部署模式**          | **集群**：集群下的所有命名空间共享一个操作员来创建和管理实例，从而降低资源消耗。                                                      |
    | **通道**              | 默认通道为 **stable**。                                                                                                               |
    | **命名空间**          | 选择 **推荐命名空间**：如果不存在则会自动创建。                                                                                       |

    4. 在 **部署操作员** 页面，选择 **默认配置**，然后单击 **部署** 完成 **Alauda 应用服务 RDS 框架** 的部署。
  </Tab>
</Tabs>
