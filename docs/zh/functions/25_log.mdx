# 日志查询

## 功能介绍
从容器层面了解实例运行过程中的日志，例如 MySQL 错误日志。妥善使用日志能帮助您快速定位问题，处理故障和异常。

## 操作步骤
<Tabs>
<Tab label="命令行操作">
1. 查看实例所有相关 pod，包含 pxc pod 和 proxysql pod。
```bash
kubectl get pod -n $namespace -l "middleware.instance/name=$name"
```
2. 查看某个 pod 日志查询
```bash
kubectl logs -n $namespace $pod
```
</Tab>

<Tab label="界面操作">

### 界面操作
1. 在左侧导航栏中，单击 **MySQL-PXC**。

2. 单击 ***命名空间的名称***。

2. 单击 ***实例名称*** 。
    
3. 在 **日志** 页签中，设置查找条件。

    * 查找特定容器组即 MySQL 副本的日志：在 **容器组** 处，选择一个容器组，并单击 **搜索**。
    
    * 查找特定时间段内的日志：在 **时间范围** 处，选择一个时段。

4. 稍等片刻，并查看结果。
</Tab>
</Tabs>
