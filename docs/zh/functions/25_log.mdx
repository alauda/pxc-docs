# 日志

了解实例运行时在容器级别的日志，如 MySQL 错误日志。正确使用日志可以帮助您快速定位问题并处理故障和异常。

## 操作步骤

<Tabs>
  <Tab label="CLI">
    1. 检查实例的所有相关 Pods，包括 pxc pod 和 proxysql pod。

    ```bash
    kubectl get pod -n <namespace> -l "middleware.instance/name=<name>"
    ```

    2. 查看特定 pod 的日志查询

    ```bash
    kubectl logs -n <namespace> <pod>
    ```
  </Tab>

  <Tab label="Web Console">
    ### 界面操作

    1. 在左侧导航栏中，单击 **MySQL-PXC**。

    2. 单击 ***命名空间名称***。

    3. 单击 ***实例名称***。

    4. 在 **实时日志** 页签中，设置查找条件。

       - 要查找特定 Pods 的日志，这些 Pods 是 MySQL 副本：在 **Pods** 下，选择一个 pod 并单击 **搜索**。

    5. 稍等片刻，查看结果。
  </Tab>
</Tabs>
