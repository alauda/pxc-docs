# 日志

日志提供了有价值的信息。检查数据库 Pods 和 Operator Pod 的日志是有意义的。以下标志在使用 kubectl logs 命令检查日志时非常有用。

## 操作步骤

<Tabs>
  <Tab label="CLI">
    1. 检查与实例相关的所有 Pods，包括 pxc pod 和 proxysql pod。

    ```bash
    kubectl get pod -n <namespace> -l "middleware.instance/name=<name>"
    ```

    2. 查看特定 pod 的日志查询

    ```bash
    kubectl logs -n <namespace> <pod>
    ```
  </Tab>

  <Tab label="Web Console">
    ### 界面操作

    1. 在左侧导航栏中，单击 **MySQL-PXC**。

    2. 单击 ***命名空间名称***。

    3. 单击 ***实例名称***。

    4. 在 **实时日志** 页签中，设置查找条件。

       - 要查找特定 Pods 的日志，这些 Pods 是 MySQL 副本：在 **Pods** 下，选择一个 pod 并单击 **搜索**。

    5. 稍等片刻，查看结果。
  </Tab>
</Tabs>
