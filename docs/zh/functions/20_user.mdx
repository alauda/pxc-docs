# 用户管理

## 功能简介
提供界面化用户管理功能，建议您根据使用需要，为数据库开发人员对应分配数据库用户。



## 前提条件

确保实例为 **运行中** 状态。


## 创建用户

创建用户并为用户配置权限。平台会同步生成供 Kubernetes 资源使用的用户身份认证凭据 -  **保密字典** 。对于使用 MySQL 的应用，可在容器组中通过保密字典的方式引用 MySQL 用户名和密码。保密字典将自动同步用户信息，请勿修改其中配置。

<Tabs>
<Tab label="界面操作">
#### 创建用户

1. 在左侧导航栏中，单击 **MySQL-PXC**。

2. 单击 ***命名空间的名称***。

3. 单击 ***实例名称*** 。

2. 在 **用户管理** 页签中，单击 **创建用户**。

4. 设置用户信息。

    **注意**：请妥善保存您的密码。为降低密码泄漏风险，建议定期更新密码。

5. 为保证用户能正常使用，请及时为用户指定权限。

6. 单击 **创建**。


#### 新增保密字典（非平台创建的用户）

如果通过非平台方式创建了数据库用户，请手动为其创建对应的保密字典。**保密字典** 是供 Kubernetes 资源使用的用户身份认证凭据。如果应用需要使用 MySQL，可在容器组中通过保密字典的方式引用 MySQL 用户名和密码。


1. 在左侧导航栏中，单击 **MySQL-PXC**。

2. 单击 ***命名空间的名称***。

3. 单击 ***实例名称*** 。

4. 在 **用户管理** 页签中，单击 ***用户名称***。

5. 填入创建用户时设置的密码，单击 **添加**。
    



#### 更新用户



1. 在左侧导航栏中，单击 **MySQL-PXC**。

2. 单击 ***命名空间的名称***。

3. 单击 ***实例名称*** 。

4. 进入 **用户管理** 页签。


    * 更新密码：单击用户右侧的 **更新密码**。
    
    * 更新用户权限（非 **root** 用户）：单击 ***用户名称***，在用户详情中，单击 **更新权限**。

5. 修改配置。

6. 单击 **更新**。
