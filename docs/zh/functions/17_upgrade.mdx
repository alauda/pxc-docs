# 补丁版本升级策略

随着漏洞修复或新功能的发布，Alauda MySQL-PXC 数据库服务将进行补丁版本升级。在这些补丁版本升级期间，Pods 将逐一重启，建议选择适当的时间窗口进行升级。当可升级的补丁版本出现时，建议尽早升级，以便将最新的修复应用到组件上。

## 设置补丁版本升级策略

### 过程

<Tabs>
  <Tab label="web 控制台">
    1. 进入实例详细信息页面。

    2. 点击 **补丁版本升级策略**。

    3. 根据需要选择 **手动** 或 **自动**。

    4. 点击 **更新** 完成设置。

    ## 补丁版本升级

    ### 前提条件

    当可升级的补丁版本可用时，实例列表中将显示升级图标，并且实例详细信息页面将出现升级提示。

    ### 步骤

    1. 进入实例详细信息页面。

    2. 点击升级提示旁的 **立即升级**。

    3. 阅读升级提示，了解组件和插件的版本变更。

    4. 点击 **立即升级**。

    5. 等待实例状态变更为运行状态，升级成功。

    ## 批量补丁版本升级

    ### 前提条件

    用户必须具有平台管理员权限。

    ### 步骤

    1. 在左侧导航栏中点击 **操作员**。

    2. 点击 **操作员** 进入操作员详细信息页面。

    3. 选择要升级的实例，并点击 **批量升级**。

    4. 阅读升级提示，了解组件和插件的版本变更。

    5. 点击 **立即升级**。

    6. 等待实例状态变更为运行状态，升级成功。
  </Tab>
</Tabs>
