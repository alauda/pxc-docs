# 创建实例

Alauda 数据库服务（MySQL-PXC）提供了一整套功能，用于创建和管理 Kubernetes 集群中的 MySQL 数据库实例。用户可以通过配置资源规格、参数模板、账户信息、调度策略和 Pods 容忍规则，快速部署符合需求的 MySQL 实例。一旦实例创建完成，部分参数可以动态调整，但调度配置不可修改。

## 前提条件

在创建 MySQL-PXC 实例之前，请确保您已查看过 <ExternalSiteLink name="acp" href="/appservice/overview/features.html#compatible_sc" children="兼容的存储类" />，并确认当前集群支持该类型的存储类，且有足够的可用空间。建议使用 TopoLVM 存储类。

## 操作步骤

<Tabs>
  <Tab label="CLI">
    1. 创建密码

    ```bash
    kubectl -n <namespace> create secret generic <name>  \
        --from-literal=clustercheck=<password> \
        --from-literal=monitor=<password> \
        --from-literal=operator=<password> \
        --from-literal=proxyadmin=<password> \
        --from-literal=root=<password> \
        --from-literal=xtrabackup=<password> 
    ```

    ::: info

    - `<name>` 是用于标识实例的密码名称，必须唯一，建议与实例使用相同名称。此设置在实例的 spec.pxc.secretsName 字段下。
    - `<namespace>` 是实例所属的命名空间。
    - `<password>` 是密码；上述提到的多个用户可以设置不同的密码。

    :::

    - `<spec.pxc.pxc.volumeSpec.persistentVolumeClaim>` 是 PXC Pods 的数据卷配置。
    - `<spec.pxc.pxc.size>` 是 PXC Pods 的数量。
    - `<spec.pxc.proxysql.resources>` 是 ProxySQL Pods 的资源配置，对应于 Pods 的资源配置。
    - `<spec.pxc.proxysql.volumeSpec.persistentVolumeClaim>` 是 ProxySQL Pods 的数据卷配置。
    - `<spec.pxc.proxysql.size>` 是 ProxySQL Pods 的数量。

    3. 等待实例创建完成

    ```bash
    kubectl get mysql -n <namespace> <name> -w
    ```

    执行该命令，等待结果中 STATE 字段变为 ready。

    4. 访问实例

    - 成功创建后，您可以访问该实例并检查连接状态。

  <Tab label="Web Console">
    1. 在左侧导航栏中，单击 **MySQL-PXC**。
    2. 单击目标 **命名空间** 的名称。
    3. 单击 **创建 MySQL-PXC 实例**。
    4. 配置相关参数，参考以下说明。

    ### 配置说明

    | 配置项      | 说明                                                         |
    | ----------- | ------------------------------------------------------------ |
    | **参数配置** | - **参数模板**：可选择系统模板或自定义模板。                                                                       |
    | **资源配置** | - **规格**：根据数据量级参考平台提供的 MySQL 性能表现，合理配置 CPU 和内存。           |
    | **账户配置** | - **管理员密码**：请妥善保存您的密码，为降低密码泄漏风险，建议定期更新密码。           |
    | **调度配置** | - **节点标签**：选择合适的标签以筛选集群中的可用节点。Pods 将被调度到可用节点。                             |
    |             | 注意：实例创建后，该配置将不可修改。                                                                                    |
    | **Pods 容忍** | - 如果节点设置了污点，只有设置了 Pods 容忍的实例才能被调度到该节点。匹配规则如下：                                 |
    |             | - **Equal**：Pods 容忍的 key、value、effect 必须与节点污点完全一致。                                                           |
    |             | - **Exists**：Pods 容忍的 key、effect 必须与节点污点的设值一致。                                                                   |
    |             | 更多信息，请参见 [Kubernetes 官方文档](https://kubernetes.io/docs/concepts/scheduling-eviction/taint-and-toleration/)。 |

    5. 单击 **创建**。

    6. 等待实例创建

    - 当实例的状态变为 **运行中** 时，表示实例创建成功。

    7. 访问实例

    - 成功创建后，您可以访问该实例并检查连接状态。
  </Tab>
