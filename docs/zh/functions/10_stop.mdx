# 停止和启动实例

## 功能简介
一键启停功能满足日常开发和运维维护的需求，方便您快速暂停或恢复中间件实例。在计划性停机、节点负载过大以及开发测试环境等场景中，您可能需要临时释放 CPU 和内存资源而不删除实例。一键启停功能帮助您节省资源并灵活管理实例，从而降低成本，提高效率。

## 前提条件
停止实例要求实例处于 **处理中**、**运行中**、**异常** 或 **未知** 状态。  

启动实例要求实例处于 **停止中** 或 **已停止** 状态。

## 注意事项
建议在停止实例前先做一次全量备份，以防数据丢失。

## 操作步骤
<Tabs>
  <Tab label="CLI">
    #### 停止实例
    ```bash
    kubectl patch mysql -n <namespace> <name> --type='merge' --patch '
    spec:
        pause: true
    '
    ```

    #### 启动实例
    ```bash
    kubectl patch mysql -n <namespace> <name> --type='merge' --patch '
    spec:
        pause: false
    '
    ```
  </Tab>

  <Tab label="Web Console">
    #### 停止实例
    1. 在左侧导航栏中，单击 **MySQL-PXC**。

    2. 单击 ***命名空间名称***。

    3. 单击 ***实例名称***。

    4. 单击 **操作** > **停止**，并确认。

    ### 停止后无法使用的功能

    | 功能                     | 说明                                                 |
    | ----------------------- | ----------------------------------------------------- |
    | **终端控制台**          | 无法使用终端控制台。                                 |
    | **操作**                | 无法重启、更新或修改调度配置。                       |
    | **拓扑**                | 无法查看拓扑页面。                                   |
    | **参数配置**            | 无法更新参数或导出参数。                             |
    | **用户管理**            | 无法进行用户管理。                                   |
    | **备份恢复**            | 无法执行备份。                                       |
    | **巡检**                | 当前实例的巡检结果无效。                             |

    #### 启动实例
    1. 在左侧导航栏中，单击 **MySQL-PXC**。

    2. 单击 ***命名空间名称***。

    3. 单击 ***实例名称***。

    4. 单击 **操作** > **启动**，并确认。
  </Tab>
</Tabs>
