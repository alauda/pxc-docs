# 重启实例

如果实例在运行过程中遇到操作故障、连接数过多或其他无法识别的未知问题，可以通过重启实例来解决这些问题。

## 注意事项

重启过程中，Pods 将会逐个重启，这可能导致服务的临时中断。

## 前提条件 

实例必须处于 运行中 状态。

**注意：**

如果实例处于 处理中 状态，表示实例可能正在正常启动；请耐心等待。

如果实例无法启动或状态异常，即实例处于 处理中 或 错误 状态，健康检查超时后，实例将会自动重启；请耐心等待。

## 操作步骤
<Tabs>
  <Tab label="CLI">
    1. 重启实例

    ```bash
    kubectl patch mysql -n <namespace> <name> --type='merge' --patch '
    spec:
        pxc:
          pxc:
              annotations:
                  percona.com/Restart-time: '$(date -u +'%Y-%m-%dT%H:%M:%SZ')'
          proxysql:
              annotations:
                  percona.com/Restart-time: '$(date -u +'%Y-%m-%dT%H:%M:%SZ')'    
    '
    ```
  </Tab>

  <Tab label="Web Console">
    1. 在左侧导航栏中，单击 **MySQL-PXC**。

    2. 单击 ***命名空间的名称***。

    3. 单击 ***实例名称***。

    4. 单击 **操作** > **重启**，并确认。
  </Tab>
</Tabs>
