# 介绍

Alauda Database for MySQL-PXC 使用同步复制的方式运行在 Kubernetes 上，基于 Percona XtraDB Cluster，它是 MySQL 的开源发行版本。Alauda Database for MySQL-PXC 提供了自动化部署与管理 Percona XtraDB Cluster 中对象的能力，也可以用来启动新的 Percona XtraDB Cluster 实例或扩展现有环境。通过 Alauda Database for MySQL-PXC，您可以轻松地部署和管理高可用的 MySQL 集群。

## 核心优势


- **自动化管理**：通过 Percona Operator 自动化管理 MySQL 集群，简化了运维工作。
- **高可用性**：基于 Percona XtraDB Cluster 提供同步复制和高可用性，确保业务的持续运行。
- **集群扩展与缩容**：支持通过简单的参数调整进行集群扩展和缩容，满足动态的负载需求。
- **自动备份**：支持定时备份，并能够将备份文件存储在 PVC 或 S3 存储中，保障数据的安全性。
- **代理支持**：集成 ProxySQL 作为 MySQL 集群的代理，自动部署和配置，提升数据库访问性能和可扩展性。
- **灵活配置**：支持参数模板，允许自定义集群配置，满足不同用户需求。

## 应用场景

- **高可用 MySQL 集群**：适用于需要高可用性的数据库系统，特别是业务关键型应用。
- **自动化数据库运维**：为企业提供自动化的 MySQL 部署、扩展、备份和恢复功能，减少人工干预。
- **分布式数据库架构**：用于构建基于 Kubernetes 的分布式数据库架构，支持弹性扩展。
- **多云与混合云环境**：支持跨多个云环境部署 MySQL 集群，实现更高的容错性和业务连续性。

## 使用限制

- **支持版本**：目前仅支持 MySQL 5.7 版本。
- **最小副本要求**：建议至少配置三副本以确保高可用性。
- **硬件要求**：部署时需确保节点具备足够的资源，以支持数据库的高性能运行。
